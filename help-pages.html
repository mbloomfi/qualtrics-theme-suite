Installation and Setup
To install QTS, please visit the GitHub and click Download Zip. After downloading the Zip file, you'll need to unzip it, then unzip the zip file within it. Finally, move the QTS app from that folder into your applications.

After firing up the app, you'll need to set your preferences before you see anything but a blank screen.
  1. Click on "QTS" in the top menu and select "Preferences"
  2. Click "Files", and you'll be presented with a filepath. This is the location of your Theme files. You'll need to fill this, and it's more than likely something like: Google Drive/Design Team/Themes
  3. Click "Save Changes" and then Quit the app (completely)
  4. Reboot the app, and you should see a screen like the following:
  5. Now click "QTS" one last time and select "Check for Updates"
Creating and Editing Themes
Creating New Brands and Themes
QTS makes building themes easy! Gone are the days of having a SASS compressor open or needing to copy and paste endless HTML files. To get started building a theme, use the Brand dropdown to search for an existing brand or create a new one. Type in the BrandID, then select the brand or click Create (or press enter).
After selecting the brand, use the Projects dropdown to either select an existing theme or create a new one. To create a new project, click the text at the bottom, type the name, then click the "+". Immediately upon creation, you'll see the default theme load on the right hand side preview pane and the SCSS variable sheet on the left. To change the file you are looking at, use the files dropdown.
Editing Themes
Editing the SCSS variables is exactly the same as you would experience within Sublime or another text editor. Upon each save (command+ s), the theme will recompress and show you the updated preview on the right. Awesome, right?
Editing Files
Although you can't make heavy duty edits to files, there are a few things you can do within QTS. If you click the files menu, you'll see 3 icons at the top-right in the menu. If you use the pencil, you can click files to edit their names (useful for logo files). If you use the red circle, then you can delete files the same way.
Apart from editing names, you can also click the image files to change their size or simply see their dimensions. This is useful when adding the sizes to the SCSS variables. You can also use the image icon to see screenshots that have been taken.

Taking Screenshots
Screenshots and Thumbnails
Screenshots are much easier within the product. To bring up the screenshot view, hit command + 2, or use the Preview dropdown in the top-menu and select Screenshot. In this screen, you can screenshot your preview. To change the width of the preview, use command + option + right or left arrow key. To get a good mobile screenshot, you can make it about the width of a phone, pull up the bottom of the screen, and then take a screenshot. A better screenshot view is on the way, but this is great as well.

One awesome thing about this approach is that QTS actually saves these screenshots in your Assets folder within the theme folder itself! No more digging through your desktop!
Thumbnail
Thumbnails are similar to Screenshots, except you use command + 3, or use the Preview dropdown in the top-menu and select Thumbnail. You can either drag and drop the screenshot box with your mouse or use your arrow keys to move it around. Enter will take the screenshot, and save it as Thumb.gif in your folder. It will always be the correct size.
Other information
Headers
Headers by default don't show up in QTS. If you want them to, go to Preview and toggle it on.
Viewing the theme in the browser
Sometimes it's necessary to view your theme in a browser. To do so, click command + shift + p. It will open in your native browser, and you can use that link in other browsers locally.
